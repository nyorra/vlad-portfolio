# Project Structure

## Directories

- `public/` — Static media assets (images, fonts).
- `src/` — Main source code directory.
  - `components/` — UI components (buttons, inputs, and other interface elements).
  - `layouts/` — Layout modules (header, footer, base `<html>` structures).
  - `pages/` — Routing directory (files here automatically deploy as pages).
  - `pages/index.astro` — Main website page (root index `://site.com`).

## Configuration

- `.env` — Local environment variables (API keys, tokens, passwords).
- `astro.config.mjs` — Main Astro config file (Tailwind, integrations, etc.).
- `tsconfig.json` — TypeScript rules and path aliases (e.g., `@components/*`).

## Dependencies

- `package.json` — Project metadata and dependency versions.
- `package-lock.json` — Autogenerated lockfile for package versions (do not modify manually).

## Documentation & Git

- `.gitignore` — Untracked files and folders ignored by Git.
- `README.md` — General project documentation.
- `about.md` — Architecture explanation and architectural decisions.
